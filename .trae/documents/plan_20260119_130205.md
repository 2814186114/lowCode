# AI上下文管理系统集成计划

## 1. 系统现状分析

### 1.1 现有AI调用流程
- **AIGeneratePanel**：直接调用`generateSchema`函数，传递会话上下文
- **PropertyPanel**：通过`APIConnector.aiGenerateSchema`调用，缺少上下文传递
- **APIConnector**：封装AI调用，但未支持会话上下文

### 1.2 集成点分析
1. **APIConnector.aiGenerateSchema**：需要支持会话上下文传递
2. **PropertyPanel.handleAiGeneration**：需要获取并传递会话上下文
3. **App组件**：需要初始化AI客户端

## 2. 集成方案设计

### 2.1 APIConnector改造

**修改内容**：
- 扩展`aiGenerateSchema`方法，支持会话上下文传递
- 保持向后兼容

**预期效果**：
- 支持传递`sessionId`和`contextHistory`参数
- 内部调用优化后的`generateSchema`函数

### 2.2 PropertyPanel改造

**修改内容**：
- 获取当前会话上下文
- 调用AI生成时传递会话上下文
- 保持现有功能不变

**预期效果**：
- 支持基于上下文的AI生成
- 保持现有UI和交互不变

### 2.3 App组件改造

**修改内容**：
- 初始化AI客户端
- 预加载模型和向量数据库
- 处理初始化错误

**预期效果**：
- 应用启动时初始化AI客户端
- 提高后续AI调用的响应速度

### 2.4 新增功能

**修改内容**：
- 添加AI状态监控
- 支持清除缓存和会话
- 添加性能监控

**预期效果**：
- 提供AI服务状态监控
- 支持手动管理会话和缓存
- 提供性能指标
